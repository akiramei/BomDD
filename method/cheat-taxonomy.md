# ずる(Cheat)の分類と記録様式 — 研究の測定器

「ずる」= **BOM由来でない産物**。BOM/工程から導けず、私(製造者)の直観・経験・原版の記憶・
未文書の判断で埋めた箇所すべて。ずるは失敗でなく**観測データ**。これを取りこぼすと研究が成立しない。

## 規律 — ずるの判定基準

> **BOM自己完結性テスト**: 「原版ソースを一度も見ていない製造者が、このBOM/工程だけで適合品を作れるか?」
> 製造中に **BOM に無い事実へ手を伸ばした瞬間が「ずる」**。手を伸ばしたら止めずに進めてよいが、必ず記録する。

### 製造装置の隔離(オラクル汚染の防壁)

リバースと製造を同一文脈で行うと、BOMが不十分でも原版の記憶で穴埋めしてしまい「気づかれないずる」が生じる。
これを防ぐため、**⑤製造は M-BOM + 工程のみを渡した別エージェント(=製造装置)に行わせ、原版ソースを与えない**。
装置が BOM だけで適合品を作れない/質問してくる箇所が、可視化された本物の手法ギャップ。

## ずるの分類(仮説 — データで精緻化する)

| 分類 | 名称 | 定義 | 対応する構想上の論点 |
|---|---|---|---|
| **C1** | 表現ギャップ | 下した判断を記録する場所がBOMに無い | as-built乖離 |
| **C2** | 暗黙知 | 経験で「正解」を知っていたが手法が符号化していない | 設計工程の自動化 |
| **C3** | 工程欠落 | 実際に必要な工程がルーティングに無い | 工程表の不完全性 |
| **C4** | 受入不能 | 「速い」「見た目が正しい」等をM-BOMが判定できない | AI非決定性・主観的受入 |
| **C5** | 粒度崩壊 | 論理部品が製造単位へ綺麗に展開できない(横断的関心) | woven not nested |
| **C6** | 手戻り | 前方流れの想定に反し上流BOMを直す必要 | as-designed↔as-built |

## ずる記録様式(cheat-log.md の1件)

```
### CHEAT-NNN  [分類] 一行要約
- 発生段: ①リバース / ②E-BOM / ③M-BOM / ④工程 / ⑤製造 / ⑥合否
- 手法が与えなかったもの: (BOMに在るべきだったが無かった情報・判断)
- 代替した従来技術: (実際に何をして埋めたか)
- 重大度: 工業化を阻む(blocker) / 不便(friction) / 軽微(minor)
- 想定される手法的修正: (次ループでBOM/工程をどう拡張すれば消えるか)
```
