# Phase 0 — チャーター(設計者 AI への指示)

典拠: `method/bomdd-playbook-v1.md` §1。テンプレ: `method/templates/00-charter.md`。

## 目的
ブレストに入る前に、**後から動かすとループ測定が濁る項目**を先に固定する。チャーターは Phase 1 以降の全フェーズの前提になる。

## 手順
`bomdd/00-charter.md` をユーザーと埋める。固定の強さは項目で違う:

**この時点で固定する(以後のループで不変)**:
- **工場構成**: ティア(最小=1工場 / 推奨=初回2–3工場→収束後1工場 / 研究=全ループ多工場)と使用モデル。途中で変えると決定性の測定が無効になる。
- **収束ループ予算**: 回数上限。打ち切り判断を後出しにしない。
- **役割**: 設計者・仕様監査リーダー数(G2)・承認者。題材に知覚/golden 特性が見込まれるなら**承認者(人間)をここで確保する**(後から探すと G2' で詰まる)。
- **黒箱境界の種別**: wire(HTTP)/ in-process(観測契約が要る)/ GUI(golden+承認者が要る)。受入設計の段取りが変わるため先に見立てる。

**仮置きでよい(Phase 1 終了時に確定)**:
- 題材の記述・スコープ(含む/含まない)。ブレスト前に細部は決められない。仮置きには `(仮)` を付け、未記入のまま進まない。

## 完了ゲート G0
- 固定項目(工場構成・予算・役割・境界種別)がすべて埋まっている。
- 完了の定義(固定オラクル全通過+blocker ずるゼロ+納品物)がテンプレ通り記載されている。

完了したら Phase 1(`phase1-brainstorm.md`)へ。
