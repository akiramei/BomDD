# /eco-fix — プローブ先行の是正+機械受入

典拠: [bomdd/change-management.md](../../../bomdd/change-management.md) §3.3。
引数: ECO 番号(例: `eco-012`)。

## 前提確認

1. `bomdd/60-change-order-eco-NNN.md` と register エントリが存在し、工程診断が
   「実装層」(または合意済みの是正対象)を指していること。無ければ /eco-file へ差し戻す。
2. 同一ファイルに触る進行中の ECO 系列がないか register で確認(diff 混濁回避)。

## 手順

1. **プローブ先行(R5)**: 是正**前**に、症状を固定する回帰テストを追加し、実行して
   **不合格になることを確認**= 真因の実測裏取り。
   - **プローブが不合格にならない場合、診断が誤り。コードに触らず /eco-file の工程診断へ差し戻す。**
   - **UI サーフェスの新設/視覚変更を含む場合**: 視覚 probe は CAD の視覚契約チェックリストから
     **この時点で生成**する(golden 所見の後追い禁止)。チェックリスト未整備の面に触れる場合は、
     先に CAD 側でチェックリストを生成してから着手(lazy 遡及)。
2. **是正の裁定**: ECO 本文 §4 の案から選ぶ。選定基準は「**真因構造そのものを消す案を優先**」
   (対症の最小 diff が真因構造を温存するなら採らない)。既存 golden の視覚不変を尊重。
   コメントは既存流儀(ECO 番号+制約の理由)で。
3. **機械受入**: CLAUDE.md の機械受入コマンドを全部回す(全緑・プローブも合格に転じること)。
   **R6: 既存固定オラクル行は変更しない。** 期待値改訂が必要になったら停止して報告(挙動保存の破れ)。
3.5. **セルフゴールデン(R7・UI サーフェスに触れた fix のみ)**: golden 提示の**前**に、是正対象の
   各サーフェスを CAD captures と並置する(headless レンダリング/実機スクリーンショット)。
   差分を**全列挙**して「裁定済み許容差分(裁定記録を引用)/転写漏れ」に分類し、**転写漏れ 0 に
   なるまで golden に出さない**。共通契約(ダイアログ言語等)に触れた場合は、直した面でなく
   **契約×適用面マトリクスの該当列の全面**を並置する。並置結果は手順 4 の実施記録へ含める。
4. **記録**: ECO 本文へ実施記録節(実測裏取り・裁定理由・diff 規模・機械受入結果)を追記し、
   register の status 注記を「是正+機械受入完了・golden 待ち」へ。
5. **コミット**: `fix(eco-NNN): <要約>`(機械受入結果を本文に含める)。

## 停止点(human gate② = golden)

**golden 合格基準を操作手順つきの箇条書きで提示して停止する。**
共有コンテナ/共有 VM を触った場合は**非表示状態(条件付き可視の裏面)の再検査**も基準に含める。
設定・永続に触れた場合は**再起動跨ぎの確認**を基準に含める。

合格の報告を受けたら /eco-accept eco-NNN へ。不合格所見(GF-*)が出たら本スキルの手順 1 から
(所見が別欠陥なら R3: 分離起票)。

## スコープ外所見(R3)

是正中に見つけた別問題は現 ECO の diff に混ぜない。分離起票(/eco-file)か 51-cheat-log 記録の二択。
