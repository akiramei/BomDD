# Migration Responsibility Matrix

| Role | Assigned person/team | Decision scope | Substitute | Evidence/approval method |
|---|---|---|---|---|
| Migration Owner | TF-001 Assigned Team | Scope, order, final Go/No-Go | TF-001 Assigned Team | Signed Gate result |
| Specification Owner | TF-001 Assigned Team | Preserve/change/legacy-defect rulings | TF-001 Assigned Team | Spec ruling record |
| DB Custodian | TF-001 Assigned Team | DB copy, permission, backup, integrity, writer control | TF-001 Assigned Team | DB evidence approval |
| WPF Technical Owner | TF-001 Assigned Team | .NET/WPF/data access/package decisions | TF-001 Assigned Team | Decision record |
| UI Approver | TF-001 Assigned Team | Display, interaction, accessibility, golden | TF-001 Assigned Team | UI approval evidence |
| Acceptance Owner | TF-001 Assigned Team | Oracle, Control Plan, Gate result | TF-001 Assigned Team | Gate approval |
| Migration Worker | TF-001 Assigned Team | Execute one assigned STEP and preserve evidence | TF-001 Assigned Team | Handoff record |

Completion rule: `UNASSIGNED` is zero before MIG-00 approval.
