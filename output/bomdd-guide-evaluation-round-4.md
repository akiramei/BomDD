## 2. G2の再評価

### 確認した条件

G2は次を明示的に要求する。

- DDD、MDD、要求トレーサビリティ、ドキュメント駆動の**四手法すべて**を対象にする。
- 各手法について、BomDDとの共通点を一つ示す。
- 各手法について、BomDDが追加する完了条件を一つ示す。
- 合計点による代替を認めない。
- 一手法でも無回答・混同、またはBomDDによる全面置換とした場合は不合格にする。

### 反例検査

| 仮の回答 | G2判定 |
|---|---|
| DDD、MDD、要求トレーサビリティだけ正答し、ドキュメント駆動を無回答 | 不合格 |
| 四手法の共通点は書くが、MDDに対するBomDD追加条件を書かない | 不合格 |
| DDDはBomDDに置き換えられるとする | 不合格 |
| 三手法で6点以上を得て、一手法の欠落を相殺する | 明示的に代替禁止のため不合格 |

したがって、Round 3で指摘したG2内部の相殺経路はない。

## 3. G3の再評価

### 確認した条件

G3は次を明示的に要求する。

- E-BOM、K-BOM、M-BOM、S-BOMの**四層すべて**を正しく区別する。
- 四層を分ける理由を、異なる問い、権威・変更理由、必要な改訂・再製造・再検査範囲などの因果で一文以上説明する。
- 一ファイルへ物理統合すること自体を誤りとしない。
- 合計点による代替を認めない。
- 一層でも無回答・混同、S-BOMをSoftware Bill of Materialsへ還元、分離理由なし、ファイル存在だけで適用済みとする回答を不合格にする。

### 反例検査

| 仮の回答 | G3判定 |
|---|---|
| E/K/Mは正答するが、S-BOMをOSS依存一覧とする | 不合格 |
| 問3の8項目を正しく配置するが、分離理由を書かない | 不合格 |
| E/K/M/Sを別ファイルにすればBomDD導入済みとする | 不合格 |
| 一つのYAMLへ統合したことだけを誤適用として挙げる | 不合格 |
| 三層の正答で一層の誤りを相殺する | 四層すべてが必須のため不合格 |

したがって、Round 3で指摘した一層誤りと分離理由欠落の相殺経路はない。

## 6. 本文・問題・模範解答の整合

最終確認で矛盾または本文未導入の必須知識を認めなかった。

- 問8は、本文で説明されたS-BOMの依存逆引き、再検査、交換/再製造、ECO再入口から回答できる。固有の`scratch`手順を要求しない。
- 問10は本文4.1〜4.3の通常AI側、BomDD側、共通部分を直接問う。通常側をテストやGitのない藁人形にする回答を落とす。
- 問4・5は本文3.2と6.5に対応し、工場セルフテストと設計者側固定オラクル、変更前ソースと変更後正解を混同させない。
- 問6は、物理的な一ファイル統合自体を誤りにせず、責務・測定器・同期の欠落を判定する。
- 問7は規模だけでなく、データ、効力、監査、失敗コストから導入可否を判断させる。
- 問9は観測範囲を一般的優越性や欠陥ゼロへ拡張する誤りを識別する。
