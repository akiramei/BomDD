# ② 変更イベント — KB-FFMPEG-SEEK-001

実 ffmpeg 更新でなく、**K-BOM の知識更新イベント**として安全に注入する(外部仕様の理解が進んだ=要求水準が上がった、というケース)。

## イベント
- **ID**: KB-FFMPEG-SEEK-001
- **対象 K-BOM**: KB-FFMPEG-SEEK を version 1 → 2 に更新
- **内容**:
  - 旧(v1): シーク時、単一シーク入力 + 音声相対時刻も許容(内容は正だが AAC のシーク由来サンプルずれの恐れ)。
  - 新(v2): **AAC/音声同期保証が必要な書き出しでは、映像=シーク入力・音声=別入力(未シーク)・絶対時刻 を必須**とする。
- **理由**: AAC のサンプル精度同期を保証するため(外部知識の精緻化)。
- **核への影響**: なし(ドメイン代数・幾何・コーデック白名簿はこの知識に依存しない)。

## このイベントが問う PLM の能力
- どの稼働部品が KB-FFMPEG-SEEK に依存するか(=再検査対象)。
- その再検査はどの深さか(L2 では不可、L3 が要る)。
- 現 build は新ポリシーに適合するか → 交換判断。
