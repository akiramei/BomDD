# Loop 6 総括 — S-BOM/PLM(表面知識の劣化と再検査計画)

三層BOMの第三層=保守層に初めて入った。K-BOM 知識更新イベント(KB-FFMPEG-SEEK-001)を**表面部品の劣化**として注入し、
S-BOM から「影響部品・再検査・交換判断」を導けるかを検証した。**眼目は新バグ発見でなく、再検査の絞り込み。**

## 成功条件の充足
| 条件 | 結果 |
|---|---|
| 1. 影響が Export 表面(シーク経路)に限定 | ✅ 6部品中1部品のみ(83%除外)。Export 非シーク経路も除外 |
| 2. 核(Split/Timeline幾何/ExportPlan)は再検査不要 | ✅ いずれも KB-FFMPEG-SEEK 非依存 |
| 3. Export は L2 でなく L3 が必要と導ける | ✅ S-BOM の受入深さ列が L3 を指定。L2 は correct/buggy を区別不能(Loop5) |
| 4. 交換不要/K-BOM更新のみ/再製造必要 を判断 | ✅ 現build=K-BOM更新のみ / buggy=再製造 / phaseA=再製造(構造) |

## 実行した再検査(S-BOM がトリガ)
S-BOM 逆引き → 影響=Export シーク経路1部品 → 受入深さ=L3 → L3治具(loop-05)を SC4 で実行。
現build(2入力・絶対時刻)は v2 ポリシー準拠かつ L3内容OK → **再製造不要、K-BOM を v2 へ更新し再認証**で完了。

## 発見
- **PLM の価値は被覆でなく絞り込み**: 素朴な「全部再検査」を、トレース(外部知識依存×受入深さ)で**1部品・1深さ・1シナリオ**へ絞れた。核を一切巻き込まない。
- **表面の劣化は核を触らず起きる**: ドメイン/幾何/白名簿は無変更のまま、シーク経路の保証だけが要求水準の上昇で劣化し得る。構想の「表面=外部知識依存」が保守の文脈で実証された。
- **治具も劣化対象(二次発見)**: 知識更新は再検査の**深さ・許容差**も上げ得る。v2 のサンプル精度同期に対し、現 L3治具(1秒/100Hz)は粗く、phaseA のサブms ドリフトを検証できない。→ 治具の精緻化(±ms 相関)か構造準拠フォールバックが要る。「治具を成果物管理」(#6)の保守版。

## 三層BOMの一周
| 層 | 役割 | 本研究での実証 |
|---|---|---|
| E-BOM | 設計(核/表面/K-BOM/グラフ) | Loop1-3 |
| M-BOM | 製造(Control Plan/FMEA/検査深さ) + 工程(隔離/マルチファクトリ) | Loop1.5-5 |
| **S-BOM** | 保守(外部知識依存の劣化→影響→再検査→交換判断) | **Loop6** |

→ BomDD の三層が、MoviePad 題材で設計→製造→保守まで**一周**した。Method v1 + S-BOM テンプレートで運用形に。
