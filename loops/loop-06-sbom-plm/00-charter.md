# Loop 6 憲章 — S-BOM/PLM(表面知識の劣化と再検査計画)

三層BOMの未踏=**保守層**。設計→製造→検査(E/M-BOM)を5ループ回したので、ここで構想が一周する。

## 仮説
> 表面部品は外部知識に依存する。外部知識が変わると、**核を変更しなくても表面の保証が劣化**する。
> S-BOM は「影響部品」「再実行すべき検査」「交換/再製造判断」を導ける。

## 注入イベント(実 ffmpeg 更新でなく K-BOM 知識更新として安全に)
**KB-FFMPEG-SEEK-001**: シーク時の音声同期ポリシー変更。
- 旧: 単一シーク入力 + 音声相対時刻も許容(内容は正だが AAC ドリフトの恐れ)。
- 新: AAC/音声同期保証が要る場合は **2入力 + 絶対時刻を必須**。
- これは「外部知識の更新で受入の要求水準が上がった」ケース。核は無関係。

## 主眼(PLM の本質)
**新バグを見つけることではなく、再検査すべき場所を正しく絞ること。** 影響しない核を巻き込まないのが価値。

## 成功条件
1. S-BOM から、変更影響が **Export 表面部品(seek 経路)に限定**される。
2. 核(Split / Timeline geometry / ExportPlan.Analyze)は **再検査不要**と判断できる。
3. Export は L2 でなく **L3 audio signal** 再検査が必要と導ける。
4. 再検査結果で **交換不要 / K-BOM更新のみ / 再製造必要** を判断できる。
