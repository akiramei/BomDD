# Loop 4 憲章 — マルチファクトリ × K-BOM(ばらつきで BOM 品質を測る)

外部から得たアイディア(BOM + 工程 + 検査 + 知識部品)を検証。最強仮説:
> AI製造の失敗は部品表不足だけでなく、製造装置の**暗黙知・治具・検査・許容差が未管理**だから。
> → BOM に加え「製造条件表(Control Plan)」「検査計画(深さ+許容差+承認者)」「知識サプライヤ管理(K-BOM)」が要る。

## 中心実験 — multi-factory acceptance(アイディア#3)
**BOM 品質を、単一装置の合格率でなく「製造装置間のばらつき」で測る。**
- 全装置一致 = BOM が強い核 / 実装差 = 暗黙知依存 / 合否差 = 受入が弱い / 全員失敗 = 工程欠落。

題材: Loop2 で暗黙知(ffmpeg慣用)が最も効いた **Export フィルタグラフ(Unit B)**。
4シナリオ(恒等/スケール/速度2x/前トリム+シーク)の filter_complex と引数を生成させ、装置間で比較する。

## 介入 — K-BOM(アイディア#1)
- **Phase A**: 挙動仕様のみ(Loop2 と同じ。内部様式は装置任せ)。
- **Phase B**: 挙動仕様 + **K-BOM(ffmpeg文法パック)**([../../method/k-bom-ffmpeg.md](../../method/k-bom-ffmpeg.md))。ラベル方式・透過綴り・補間フラグ・エンコーダ・**音声絶対時刻(座標系)**・setpts形・数値書式・引数順を明示。
- 装置 = **3モデル(opus / sonnet / haiku)= 3工場**(工程能力比較)。同一プロンプト。

## 測定
- **ばらつき**: シナリオ×次元(ラベル/スケールフラグ/エンコーダ/音声時刻基準/setpts形/数値書式/引数順)ごとに、3工場の一致/不一致を集計。
- **仮説 H10**: Phase A は次元の多くで工場間ばらつき大(暗黙知依存)。Phase B(K-BOM)でばらつきが大きく縮小(管理された知識へ)。
- **仮説 H11(FMEA連動, #5)**: 事前列挙した故障モード(特に CHEAT-008 音声座標系)が、Phase A では一部工場で再発、Phase B では全工場で消える。

## Control Plan(アイディア#2)— 受入の正式列
各 AC に `検査深さ(L0文字列/L1存在/L2メタデータ/L3信号/golden) + 許容差 + 承認者` を付す。本ループの受入は
[../../method/control-plan.md](../../method/control-plan.md) に定義し、Export 表面は L2(ffprobe)+座標系は L3(信号) を要する。
