# Loop 4 総括 — マルチファクトリ × K-BOM(ばらつきで BOM 品質を測る)

外部アイディア(BOM+工程+検査+知識部品)を検証。3工場(opus/sonnet/haiku)に同じ Export フィルタグラフ仕様で
4シナリオを生成させ、(A) K-BOMなし / (B) K-BOMあり で**工場間ばらつき**を測った。

## 結果
### Phase A(K-BOMなし)= 約8次元で総当たり分岐
| 次元 | opus | sonnet | haiku |
|---|---|---|---|
| 区間ラベル | v0/a0 | v0/a0 | v1/a1(1始まり) |
| 連結構造 | concat省略→null直結 | concat=n=1→[vout][aout] | concat=n=1→[vout][aout] |
| setpts形 | 1.0*(PTS-STARTPTS) | PTS-STARTPTS | PTS/2(STARTPTS省略=潜在バグ) |

(`[1:a]atrim=start=5:end=10`)で **CHEAT-008 を全工場で修正**。エンコーダも原版どおり統一。

| 指標 | Phase A | Phase B |
|---|---|---|
| 分岐次元 | ~8 | **0** |
| 工場固有バグ | haiku 2件 | 0 |
| CHEAT-008(音声座標系) | 全工場で誤 | 全工場で正 |

## 仮説
- **H10 ✅(強)**: K-BOM が工場間ばらつきを「全次元分岐」→**ゼロ**へ。未管理の暗黙知を管理知識へ変換した量=ばらつき消失で実測。
- **H11 ✅**: FMEA で狙った故障モード(音声座標系)が、Phase A では全工場で誤、Phase B では全工場で正。K-BOM が捕捉・除去。
- **工程能力の均等化**: 低能力工場(haiku)も K-BOM で他と同一の正しい出力へ。ばらつきは工場差でなく**BOMの強さ**に支配される。

## 最重要の発見 — 「一致」には2種類ある(#3 の精緻化)
**マルチファクトリの一致は「BOMの決定性」を測るが「正しさ」は保証しない。**
Phase A の SC4 で3工場は**互いに一致**したが、それは原版と違う**共有された誤り**(単一入力・相対時刻=AACシークのサンプルずれを招く)だった。
- 一致の2型: **強いBOM核(正しく一致)** と **共有暗黙知の罠(揃って間違う)**。
- 「全工場一致」だけでは後者を見抜けない。**ばらつき指標(決定性)と execution受入(正しさ)は相補的**で、両方必要。
- これは隔離装置(防壁)の発展形: 単一装置の「気づかれないずる」を、複数装置でも**共有された**ずるは消えない。だから execution が要る。

## 評価指標としての結論(BOM品質の二軸)
ソフトウェア工業化の BOM 品質は2軸で測る:
1. **決定性 = 工場間ばらつき**(multi-factory variance)。低いほど BOM が出力を一意に決める。
2. **正しさ = execution/golden 受入**(Loop2.5/3)。低ばらつきでも誤りはあり得る。
K-BOM は (1) を劇的に上げる(暗黙知の管理化)。(2) は別途 Control Plan で担保する。
