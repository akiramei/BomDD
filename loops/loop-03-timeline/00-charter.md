# Loop 3 憲章 — タイムライン描画(UI/知覚)スライス

## 目的
未着手の CHEAT-006(知覚的受入の検査不能)に正面から。`TimelineControl`(自前描画)を題材に、
**幾何(純粋ロジック)/設計トークン(知識)/描画(知覚)** の3層で BOM 手法がどこまで効くかを測る。
自己完結のため Avalonia/Skia/フォントを使わず、**決定的な RGBA ラスタ**(整数矩形塗り・反AAなし)で golden 比較する。

## スコープ(2ユニット)
- **Unit G `TimelineLayout`**(純粋幾何): `X(t)=t*px`、区間矩形 `(X(Start),TrackTop,X(End)-X(Start),TrackH)`、
  境界 x、再生ヘッド x、トリムマスク左右、`NiceStep`(目盛間隔)とルーラー目盛。出典: TimelineControl.cs:207-209,323,364-377,566-705,892-897。
- **Unit R `TimelineRaster`**(描画): 固定キャンバスへ z-order で矩形塗り → RGBA バッファ。
  z-order(出典 :407-419,566-757): 背景→トラック地→**速度ティント**→除外マスク→トリムマスク→境界線→再生ヘッド。

UI 入力(ポインタ/ドラッグ/ホバー)・スクロール仮想化・サムネ/波形/テキストラベル・フォントは対象外。

## 実験変数(意図的)
- Unit G の規則・レイアウト定数・`NiceStep` の steps配列 は **完全規定**(industrializable 対照)。
- Unit R のトークンは **大半を規定**(surface/excluded/trimMask/boundary/playhead 色、TrackTop/TrackH、再生ヘッド幅)
  **が、速度→ティント色だけ未規定**(原版は `SpeedColor`+OKLCH の設計トークン=知識)。装置に発明させ、知覚ギャップを露出。

## 受入(golden image, 決定的ラスタの画素差分)
検査者が原版トークン+選定ティントで golden RGBA を作り、製造物 RGBA と**画素一致**で比較。
- 規定トークン領域=画素一致(幾何・指定色が正しい)を期待。
- 未規定の速度ティント領域=乖離(golden が失敗検出)を期待。差分が**速度ティント矩形に局在**するかを測る。

## 仮説
- **H7**: 幾何は規則から再生産でき単体テスト緑。ただし `NiceStep`(target=70/px・steps配列)やレイアウト定数は要求から導けない C2。
- **H8**: 規定トークン→画素一致 / 未規定トークン(速度色)→乖離。golden は「妥当な設計差」を失敗扱い=**人間が golden を authoring/承認**せねばならない=CHEAT-006 の核(要求由来の知覚オラクルは存在しない)。
- **H9**: golden image は過剰結合/被覆不足の二重性(反AA・1px インセット・描画順に過敏 / 微小レイアウトずれに鈍感)。本ループは決定的ラスタで「妥当な設計差への過剰結合」側を切り出して実証。
