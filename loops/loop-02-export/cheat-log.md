# ずる台帳 — Loop 2 (書き出しスライス)

分類は [../../method/cheat-taxonomy.md](../../method/cheat-taxonomy.md)。

### CHEAT-007  [C2 暗黙知 / 外部知識] Unit A の規則は要求から導けず、BOM が外部/業務知識を運んで初めて成立
- 発生段: ①リバース/③M-BOM
- 手法が与えなかったもの(を、今回は明記して回避): コピー白名簿の正確なコーデック綴り({H.264,H.265},{AAC,MP3,AC-3})、無音声番兵 `"—"`、各閾値(0.001/0.05/1e-4)、スパン縮退規則、そして **「VFR はコピーを阻害しない」**。
- 観測: 製造装置は「VFR 注記が無ければ逆(再エンコード必須)に実装していた」と自己申告。Unit A が industrialize できたのは、これら**外部/業務知識を BOM が運んだから**であり、規則が要求から導けたからではない。
- 重大度: friction(だが本質的): 外部ツール連携の「設計」とは、実質**外部仕様の知識を BOM へ転記する作業**。導出ではない。
- 想定される手法的修正: E-BOM に「外部依存知識(コーデック対応表・番兵・閾値)」を**明示部品**として持たせ、出典(ツール仕様)へトレース。要求からの導出と区別する。

### CHEAT-008  [C2 暗黙知 → 機能バグ] seekBase 2入力のタイミング関係が未規定で、装置が音声時刻を取り違え
- 発生段: ⑤製造/⑥合否
- 手法が与えなかったもの: 「映像入力をシークしたら、音声は別入力(シークなし)から**絶対時刻**で取る」という関係。behavioral M-BOM は「映像シークを使ってよい」までしか書かず、音声側の時刻基準を欠いた。
- 代替した従来技術: 装置は seekBase オフセットを音声 atrim にも適用(`start=0:end=5`)→ 音声が誤位置から切り出される実バグ。隠しオラクル B08 が捕捉。原版を参照し音声を絶対時刻 `s/e` に修正(1行)。
- 重大度: blocker(無音ずれの実害)。**共有 ffmpeg 知識が尽きた所で、ずるが「様式差」でなく「機能バグ」になった。**
- 想定される手法的修正: 外部ツール連携の M-BOM は、入力⇔時刻基準の対応(座標系)を明示必須化。挙動仕様に「不変条件(音声=絶対時刻)」を持たせる。

### CHEAT-009  [C4 受入不能] 文字列照合オラクルは過剰結合かつ被覆不足で、外部ツール成果物の受入に不適
- 発生段: ⑥合否
- 手法が与えなかったもの: 外部ツールが生成するコマンドの「正しさ」を判定する適切なモダリティ。
- 観測(30件中):
  - **過剰結合の誤検出4件**: B01(連結ラベル `cv` 期待 vs 製造物 `v0`+単一区間concat省略)、B10/B11/B12(空白無しパスを引用しない様式 → InputCount ヘルパが0)。**製造物は機能的に正しい**のに失敗扱い。
  - **沈黙した意味乖離(未被覆)**: Lossless が原版 `ffv1`/`pcm_s16le` に対し製造物 `libx264 -qp 0`/`flac`(別コーデック=別成果物)、scale `lanczos`→`bicubic`、setpts の綴り差。いずれもテストが assert せず通過。
- 重大度: blocker(工業化の根幹): 外部ツール成果物は、文字列照合では「任意様式の差」を誤検出し「意味的差」を見逃す。
- 想定される手法的修正: 受入モダリティを `execution`(ffmpeg を実行し、出力メディアを基準と比較=尺・解像度・PSNR/音声相関等)に格上げ。文字列照合は補助に降格。M-BOM のコーデック等「成果物の意味」は明示 AC 化。
- **解決(Loop 2.5 で実証, [../loop-02.5-export-exec/report.md](../loop-02.5-export-exec/report.md))**: execution 受入ハーネスで実測。過剰結合の誤検出は消滅(S1/S2/S4 PASS)、沈黙乖離は捕捉(S5 Lossless が ffv1/pcm 要求に対し h264/flac で FAIL)。さらに execution 受入は**深さの梯子**(L0文字列<L1存在<L2メタデータ<L3内容/信号)で、B08級の内容バグは L3 が要ると判明。M-BOM の受入は AC ごとに必要な深さを指定する。
