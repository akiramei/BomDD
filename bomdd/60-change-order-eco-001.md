# ECO-001 — 配布テンプレ 60-change-register.yaml の YAML 不正(fail-late)

## 起票(2026-07-10)

- 出典: メンバー外部レビュー所見1(P1)。「新規プロジェクトへ配布される変更台帳が YAML として不正。
  `bomdd-init` はこのファイルを必ずコピーし、/eco-file 等は台帳を前提とするため、初回 ECO で停止する」。
- 再現(是正前): PyYAML 全数パースで **14/15 合格**、`method/templates/60-change-register.yaml` のみ
  `mapping values are not allowed here`(25 行目)。原因= プレースホルダー
  `<反証可能な形の「影響なし」予測(例: 「Api 層は…」)…>` 内の**未引用のコロン+空白**。
- 工程診断: transfer 3 ラウンドで顕在化しなかったのは担当者 AI が台帳を全文書き直すため —
  **fail-closed でなく fail-late**。機械検査(yaml.safe_load を使う bomdd-lint R-051・
  impact-retrospective)はテンプレ状態の台帳で即死する。

## 裁定

- ユーザー承認(2026-07-10)「1から着手して」— レビュー所見の是正順序 1 番として実施。
- 是正方針: 該当値の二重引用符囲みのみ(最小変更)。他プレースホルダーの一括引用や
  テンプレ構文 lint の常設は**所見6(自己適合ゲート)のスコープ**とし、ここでは行わない。

## 影響分析(製造前凍結)

- 影響なし予測: `method/templates/60-change-register.yaml` の 1 行のみ — 他ファイルは diff ゼロ。
- 意味論影響: プレースホルダー文言は不変(引用符の付加のみ)。テンプレ利用側の手順に変更なし。

## 教訓(還元候補 — lesson-promote 経由)

- 配布物の構文適合は配布時でなく**リポの自己検査で常時**担保すべき(所見6の期待結果
  manifest+self-conformance gate に接続)。テンプレは「AI が読んで書き直す」経路に
  救済されて構文欠陥が潜伏する — 機械読取り経路(lint・治具)を陽性対照に含める。
